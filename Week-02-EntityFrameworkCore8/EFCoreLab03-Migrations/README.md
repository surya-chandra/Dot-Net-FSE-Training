- Rolling back migrations

## How to Run

### Step 1 вЂ” Restore packages

```bash
cd EFCoreLab03-Migrations
dotnet restore
```

### Step 2 вЂ” Create the migration (already included)

```bash
dotnet ef migrations add InitialCreate
```

### Step 3 вЂ” Apply migration to create the database

```bash
dotnet ef database update
```

### Step 4 вЂ” Run the application

```bash
dotnet run
```

---

## Migration Commands Reference

```bash
# Create a new migration
dotnet ef migrations add InitialCreate

# Apply all pending migrations
dotnet ef database update

# List all migrations
dotnet ef migrations list

# Remove the last unapplied migration
dotnet ef migrations remove

# Revert all migrations
dotnet ef database update 0

# Drop the database
dotnet ef database drop
```

## Understanding Migration Files

### `InitialCreate.cs`
- `Up()` вЂ” creates Categories, Suppliers, Products tables with indexes and FKs
- `Down()` вЂ” drops all tables in reverse order

### `ApplicationDbContextModelSnapshot.cs`
- Auto-generated by EF Core Tools
- Represents the **current state** of the model
- Used to diff against your entities when creating the next migration
- **Never edit manually**
